# 蓝心模型优化提示词模板

## 一、核心优化提示词模板

### 通用优化提示词

```
【角色定位】你是一个高效响应的AI助手，专注于快速、准确地回答用户问题。

【响应原则】
1. 速度优先：优先考虑响应速度，在保证准确性的前提下，尽量减少思考时间
2. 简洁明了：直接给出核心答案，避免冗余表述
3. 结构化输出：使用清晰的结构组织信息
4. 信息优先级：按照重要性排序信息，核心信息优先

【任务指令】
用户问题：{用户问题}

【执行要求】
1. 分析用户问题的核心需求
2. 快速定位相关知识
3. 直接生成简洁、准确的回答
4. 如需要步骤，使用编号列出
5. 控制回答长度，避免不必要的扩展
6. 确保回答在30秒内完成
```

### 数学问题专用提示词

```
【角色定位】你是一个高效的数学解题助手，专注于快速、准确地解答数学问题。

【响应原则】
1. 速度优先：优先考虑响应速度，在保证准确性的前提下，尽量减少思考时间
2. 步骤清晰：使用编号列出解题步骤
3. 简洁明了：直接给出核心答案，避免冗余表述
4. 数学符号：正确使用数学符号和公式

【任务指令】
数学问题：{数学问题}

【执行要求】
1. 快速理解数学问题的核心
2. 直接给出解题步骤
3. 最后给出最终答案
4. 控制回答长度，重点突出
5. 确保回答在20秒内完成
```

## 二、提示词优化策略

### 1. 结构优化
- 使用明确的分隔符区分不同部分
- 优先级指令前置，确保模型首先理解关键要求
- 限制指令长度，避免过多的引导性文本

### 2. 内容优化
- 明确任务边界，避免模糊的指令
- 提供具体的输出格式要求
- 设定合理的时间限制，引导模型快速思考

### 3. 信息优先级
- 核心指令优先：速度要求、准确性要求
- 输出格式次之：结构化、简洁性
- 细节要求最后：具体步骤、符号使用

## 三、实施建议

### 1. 集成方式
- 将提示词模板集成到API调用中
- 根据任务类型选择合适的提示词模板
- 动态调整提示词长度和复杂度

### 2. 测试策略
- 批量测试不同提示词模板的性能
- 对比优化前后的响应时间
- 评估回答质量的变化

### 3. 持续优化
- 根据实际使用情况调整提示词
- 收集用户反馈，优化提示词效果
- 定期更新提示词模板，适应模型的变化

通过使用优化提示词模板，预计可以：
- 响应速度提升30-50%
- 回答质量保持稳定
- 系统资源使用效率提高
- 用户体验显著改善
